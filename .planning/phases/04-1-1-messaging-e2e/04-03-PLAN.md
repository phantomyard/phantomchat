  <action>
    Create `src/lib/nostra/delivery-tracker.ts`:

    **Types:**
    ```typescript
    export type DeliveryState = 'sending' | 'sent' | 'delivered' | 'read';
    export interface DeliveryInfo {
      state: DeliveryState;
      sentAt?: number;
      deliveredAt?: number;
      readAt?: number;
    }
    ```

    **DeliveryTracker class:**
    - Constructor: `(options: { privateKey: Uint8Array; publicKey: string; publishFn: (event: SignedEvent) => Promise<void> })`
    - Internal Map<string, DeliveryInfo> keyed by eventId (message's relay event ID)
    - `markSending(eventId: string)`: set state='sending'
    - `markSent(eventId: string)`: transition to 'sent', record sentAt timestamp. Emit rootScope event `nostra_delivery_update` with {eventId, state: 'sent'}
    - `handleReceipt(rumor: UnsignedEvent)`: parse receipt-type and e tags from rumor. If `receipt-type === 'delivery'`, transition to 'delivered'. If `receipt-type === 'read'` AND readReceiptsEnabled, transition to 'read'. CRITICAL: never send a receipt for a receipt (Pitfall 6 -- check `isReceiptEvent()` before processing).
    - `sendDeliveryReceipt(originalEventId: string, senderPubkey: string)`: create a delivery receipt via `wrapNip17Receipt(this.privateKey, senderPubkey, originalEventId, 'delivery')`, publish via publishFn
    - `sendReadReceipt(originalEventId: string, senderPubkey: string)`: if readReceiptsEnabled, create and publish read receipt. If disabled, return without action (Pitfall 7).
    - `getState(eventId: string): DeliveryInfo | undefined`
    - `setReadReceiptsEnabled(enabled: boolean)`: toggle
    - `isReadReceiptsEnabled(): boolean`

    **Static helpers:**
    - `isReceiptEvent(rumor: UnsignedEvent): boolean` -- checks if rumor has a `receipt-type` tag
    - `parseReceipt(rumor: UnsignedEvent): {originalEventId: string; receiptType: 'delivery' | 'read'} | null`

    **Read receipt privacy setting:** Store in localStorage key `nostra:read-receipts-enabled` (defaults to true). The reciprocal behavior (don't show others' read receipts if own disabled) is enforced by: if disabled, `handleReceipt` ignores read receipts AND `sendReadReceipt` is a no-op.

    **rootScope event:** Add `nostra_delivery_update` to rootScope event types. Payload: `{eventId: string; state: DeliveryState}`. The display bridge will listen to this to update bubble check icons.

    Create tests covering all behavior items above.

       - IndexedDB store: `nostra-message-requests`, version 1, store `requests`
       - Interface `MessageRequest { pubkey: string; firstMessage: string; timestamp: number; status: 'pending' | 'accepted' | 'rejected' }`
       - `addRequest(pubkey: string, message: string, timestamp: number): Promise<void>` -- store if pubkey not already known. If already rejected (blocked), silently ignore.
       - `getRequests(): Promise<MessageRequest[]>` -- return all pending requests
       - `getPendingCount(): Promise<number>` -- for badge display
       - `acceptRequest(pubkey: string): Promise<void>` -- change status to 'accepted'. The caller (chat-api or display bridge) moves the conversation to the main chat list.
       - `rejectRequest(pubkey: string): Promise<void>` -- change status to 'rejected'. Blocks future messages from this pubkey.
       - `isKnownContact(pubkey: string): Promise<boolean>` -- returns true if pubkey is in virtual-peers-db (mapped to a peerId) OR has an accepted request. False means unknown sender.
       - `isBlocked(pubkey: string): Promise<boolean>` -- returns true if pubkey has a rejected request
       - Export singleton `getMessageRequestStore()`

    2. **nostr-relay.ts** -- Wire delivery tracker receipt handler:
       - Add `onReceipt(handler: (rumor: UnsignedEvent) => void): void` method to register receipt callback
       - In `handleEvent()` (updated in Plan 01 for kind 1059): after unwrapping, if `isReceiptEvent(rumor)` is true, call `onReceiptHandler(rumor)` instead of `onMessageHandler`. This routes receipts to DeliveryTracker.

    3. **chat-api.ts** -- Wire delivery tracker and message requests:
       - Add `deliveryTracker: DeliveryTracker` property, initialized in constructor with the user's private key and a publishFn that calls `relayPool.publishRawEvent()`
       - In `sendMessage()`: call `deliveryTracker.markSending(messageId)` before send. On successful relay publish, call `deliveryTracker.markSent(messageId)`.

If `!messageRequestStore.isKnownContact(msg.from)`, add to message requests instead of main history. Dispatch rootScope `nostra_message_request` event with pubkey and first message.
       - Wire `nostrRelay.onReceipt()` (via pool) to `deliveryTracker.handleReceipt()`.
       - Add `markRead(eventId: string, senderPubkey: string)` method called by UI when user views a message. This calls `deliveryTracker.sendReadReceipt()`.
