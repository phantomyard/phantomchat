    Create `src/lib/nostra/media-crypto.ts` using Web Crypto API (no npm dependencies):

    ```typescript
    export interface EncryptedMedia {
      encrypted: ArrayBuffer;
      key: Uint8Array;    // 32 bytes, raw AES key
      iv: Uint8Array;     // 12 bytes, GCM nonce
    }
    ```

    - `encryptMedia(file: ArrayBuffer): Promise<EncryptedMedia>` -- generate AES-256-GCM key via `crypto.subtle.generateKey()`, generate 12-byte IV via `crypto.getRandomValues()`, encrypt via `crypto.subtle.encrypt({name: 'AES-GCM', iv}, key, file)`, export key via `crypto.subtle.exportKey('raw', key)`. Return encrypted buffer + raw key bytes + IV.

    - `decryptMedia(encrypted: ArrayBuffer, keyBytes: Uint8Array, iv: Uint8Array): Promise<ArrayBuffer>` -- import key via `crypto.subtle.importKey('raw', keyBytes, {name: 'AES-GCM'}, false, ['decrypt'])`, decrypt via `crypto.subtle.decrypt({name: 'AES-GCM', iv}, key, encrypted)`.

    - `sha256Hex(data: ArrayBuffer): Promise<string>` -- compute SHA-256 hash via `crypto.subtle.digest('SHA-256', data)`, return as hex string. Used for Blossom blob addressing.

    - `bytesToHex(bytes: Uint8Array): string` and `hexToBytes(hex: string): Uint8Array` -- utility for key/IV serialization in NIP-17 tags.

    Create tests in `src/tests/nostra/blossom-media.test.ts` covering the behavior above. Use `crypto.subtle` from jsdom (Vitest environment). If `crypto.subtle` is unavailable in test env, use the `@noble/hashes` SHA-256 for test verification only.

    Create `src/lib/nostra/blossom-client.ts`:

    **Constants:**
    ```typescript
    export const DEFAULT_BLOSSOM_SERVERS = [
      'https://blossom.primal.net',
      'https://cdn.satellite.earth',
      'https://nostrmedia.com'
    ];
    export const MAX_PHOTO_SIZE = 10 * 1024 * 1024;   // 10MB
    export const MAX_VIDEO_SIZE = 50 * 1024 * 1024;    // 50MB
    ```

    **BlobDescriptor interface:**
    ```typescript
    export interface BlobDescriptor {
      url: string;
      sha256: string;
      size: number;
      type: string;
      uploaded: number;
    }
    ```

    **BlossomClient class:**
    - Constructor: `(options: { servers?: string[]; fetchFn?: typeof fetch })` -- `fetchFn` defaults to `window.fetch.bind(window)`. When Tor is active, pass the PrivacyTransport's fetch wrapper. `servers` defaults to DEFAULT_BLOSSOM_SERVERS.
    - `setFetchFn(fn: typeof fetch)` -- update fetch function at runtime (when Tor mode changes)
    - `upload(data: ArrayBuffer, mimeType: string): Promise<BlobDescriptor>` -- tries each server in order. For each server:
      1. `PUT ${serverUrl}/upload` with `Content-Type: application/octet-stream` body
      2. If response ok, parse JSON BlobDescriptor and return
      3. If response error or network error, log warning and try next server
      4. If all servers fail, throw Error with details
    - `download(sha256: string): Promise<ArrayBuffer>` -- tries each server in order:
      1. `GET ${serverUrl}/${sha256}`
      2. If response ok, return arrayBuffer()
      3. If not found or error, try next server
      4. If all fail, throw
    - `validateSize(size: number, type: 'photo' | 'video'): void` -- throws if size exceeds limit per user decision (10MB photos, 50MB videos)

    **Upload helper function (convenience):**
    ```typescript
    export async function uploadEncryptedMedia(
      file: ArrayBuffer,
      mimeType: string,
      client: BlossomClient
    ): Promise<{descriptor: BlobDescriptor; key: Uint8Array; iv: Uint8Array; originalSha256: string}>
    ```
    This combines: sha256Hex(original) -> encryptMedia(file) -> client.upload(encrypted) -> return all metadata needed for NIP-17 kind 15 tags.

    Import `encryptMedia` and `sha256Hex` from `./media-crypto`.

    **Download + decrypt helper:**
    ```typescript
    export async function downloadDecryptedMedia(
      sha256: string,
      keyHex: string,
      ivHex: string,
      client: BlossomClient
    ): Promise<ArrayBuffer>
    ```
    This combines: client.download(sha256) -> decryptMedia(blob, hexToBytes(keyHex), hexToBytes(ivHex)) -> return decrypted.

    Note: The Tor fetch integration depends on PrivacyTransport from Phase 3. The BlossomClient accepts any fetch-compatible function. In production, pass `privacyTransport.createFetchFn()` or equivalent. For now, the client is transport-agnostic -- it just needs a `fetch`-like function.
