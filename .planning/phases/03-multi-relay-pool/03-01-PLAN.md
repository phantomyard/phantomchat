    REWRITE `src/lib/nostra/privacy-transport.ts`:

    The new PrivacyTransport wraps the entire NostrRelayPool (not a single NostrRelay). It manages a single shared WebtorClient instance for all relay connections.

    Key changes from current implementation:
    - Remove PeerTransport dependency (no WebRTC in v1 per architecture decision)
    - Remove single NostrRelay — use NostrRelayPool instead
    - Single shared WebtorClient for all relays in pool

    New class structure:
    ```
    export class PrivacyTransport {
      private relayPool: NostrRelayPool;
      private webtorClient: WebtorClient;
      private state: 'bootstrapping' | 'active' | 'direct' | 'failed' | 'offline';
      private offlineQueue: OfflineQueue;

      constructor(relayPool: NostrRelayPool, offlineQueue: OfflineQueue)
      async bootstrap(): Promise<void>  // fire-and-forget Tor init
      confirmDirectFallback(): void     // user confirmed, switch pool to direct
      retryTor(): Promise<void>         // retry Tor after failure
      async send(recipientPubkey: string, plaintext: string): Promise<PublishResult>
      getState(): PrivacyTransportState
      disconnect(): void
    }
    ```

    bootstrap() behavior:
    1. Set state to 'bootstrapping', dispatch nostra_tor_state via rootScope
    2. Initialize relayPool (connectAll via WebSocket initially — app is interactive)
    3. Fire-and-forget: create WebtorClient, call bootstrap(60000)
    4. On success: call relayPool.setTorMode(fetchFn), set state to 'active', dispatch event
    5. On failure: set state to 'failed', dispatch event. Do NOT auto-fallback. Do NOT switch to direct. User must call confirmDirectFallback().

    send() behavior:
    - If state is 'bootstrapping' or 'failed': queue message via offlineQueue, dispatch nostra_message_queued with status 'queued'
    - If state is 'active' or 'direct': publish via relayPool.publish(), dispatch nostra_message_queued with status 'sent'

    confirmDirectFallback():
    - Call relayPool.setDirectMode() on all relays
    - Set state to 'direct', dispatch nostra_tor_state
    - Flush offlineQueue

    retryTor():
    - Set state to 'bootstrapping'
    - Retry WebtorClient bootstrap
    - On success: switch pool to Tor mode, flush queue
    - On failure: set state to 'failed' again

    IMPORTANT: Import rootScope and dispatch events for all state changes. Do NOT import PeerTransport or use WebRTC. Use path aliases. Follow code style.

    Create tests:

    - `src/tests/nostra/tor-bootstrap.test.ts` — test that bootstrap is non-blocking, state transitions emit rootScope events, app is interactive during bootstrap
