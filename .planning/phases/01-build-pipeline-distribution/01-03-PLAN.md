<task type="auto">
  <name>Task 2: Add GitHub Pages SPA 404 fallback</name>
  <files>public/404.html</files>
  <action>
Create public/404.html that handles GitHub Pages deep-route 404s by redirecting to the root while preserving the original URL in sessionStorage.

Use this standard community pattern (lightweight, no dependencies):

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Nostra.chat</title>
    <script>
      // GitHub Pages SPA fallback
      // Preserve the full path in sessionStorage before redirecting to root
      var path = window.location.pathname + window.location.search + window.location.hash;
      sessionStorage.setItem('redirect', path);
      window.location.replace('/');
    </script>
  </head>
  <body></body>
</html>
```

Note: The index.html does NOT need a corresponding "read from sessionStorage" script — Nostra.chat is a PWA that initializes from root and uses client-side pushState navigation. The sessionStorage value can be read by the app router if needed in a future phase. For Phase 1, this prevents GitHub Pages from showing its own error page on hard refresh.
  </action>

  </verify>
  <done>public/404.html exists with sessionStorage redirect pattern. After `pnpm build`, both public/_headers and public/404.html appear in dist/.</done>
