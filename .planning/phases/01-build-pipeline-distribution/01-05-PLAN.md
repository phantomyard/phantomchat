ipshipyard/ipfs-deploy-action@v1 parameters:
```yaml

  with:
    path-to-deploy: dist
    pinata-jwt-token: ${{ secrets.PINATA_JWT_TOKEN }}
    pinata-pinning-url: 'https://api.pinata.cloud/psa'

```

The action:
1. Creates a CAR file from dist/
2. Uploads to Pinata (pins the content)
3. Reports the CID as a GitHub commit status (visible in PR/commit view)
4. CID accessible via: https://ipfs.io/ipfs/<CID>/

IPFS gateway subdirectory path: /ipfs/<CID>/
Service worker already uses scope: './' (relative) — correct for gateway subdirectories.
Manifest already has start_url: "./" — correct.

Note: ENS/HNS decentralized domain integration is deferred to a future phase (per user decision in CONTEXT.md). The HTTP gateway link satisfies DIST-04 for Phase 1.

Deploy job to ADD to existing .github/workflows/deploy.yml
(alongside the existing deploy-cloudflare and deploy-github-pages jobs):
```yaml
deploy-ipfs:
  needs: build
  runs-on: ubuntu-latest
  steps:
    - uses: actions/download-artifact@v4
      with:
        name: dist
        path: dist/
    - uses: ipshipyard/ipfs-deploy-action@v1
      with:
        path-to-deploy: dist
        pinata-jwt-token: ${{ secrets.PINATA_JWT_TOKEN }}
        pinata-pinning-url: 'https://api.pinata.cloud/psa'
        github-token: ${{ github.token }}
```
