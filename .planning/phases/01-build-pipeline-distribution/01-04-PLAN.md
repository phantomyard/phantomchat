<objective>
Create the GitHub Actions CI/CD workflow that builds Nostra.chat once and deploys to Cloudflare Pages and GitHub Pages in parallel. This satisfies DIST-03: the same build artifact serves from two distinct mirror domains.

Purpose: Manual deployment is not sustainable for censorship-resistant distribution. Every push to main should automatically update all mirrors. Building once ensures all mirrors serve identical content (same asset hashes, same bundle).

Output: .github/workflows/deploy.yml that triggers on push to main, builds dist/, uploads as an Actions artifact, then deploys to Cloudflare Pages and GitHub Pages in parallel.

Workflow pattern (single build, parallel deploys):
  job: build → upload dist/ as artifact
  job: deploy-cloudflare (needs: build) → download artifact → wrangler-action pages deploy
  job: deploy-github-pages (needs: build) → download artifact → deploy-pages

Anti-patterns from research (DO NOT USE):
  - cloudflare/pages-action (DEPRECATED — use wrangler-action@v3)
  - Building in each deploy job (produces different hashes, defeats purpose)
  - base: 'https://nostra.chat/' in vite.config.ts (locks build to one origin)

Cloudflare Pages wrangler-action command:
  command: pages deploy dist --project-name=nostra

GitHub Pages requires these permissions on the deploy job:
  permissions:
    pages: write
    id-token: write
