       - Import `importFromMnemonic` from `@lib/nostra/nostr-identity`
       - Import `generateBrowserScopedKey, encryptKeys, saveEncryptedIdentity, saveBrowserKey` from `@lib/nostra/key-storage`
       - Import `loadIdentity, clearIdentity` from `@lib/nostra/identity` (old module — reads existing OwnID data)
       - Import `loadAllQueuedMessages, saveQueuedMessage` from `@lib/nostra/offline-queue`
       - Import `nip44Encrypt, getConversationKey` from `@lib/nostra/nostr-crypto`
       - Import `hexToBytes` from `@noble/hashes/utils`
       - Import rootScope from `@lib/rootScope`

       - Export `async function needsMigration(): Promise<boolean>`:
         Read old identity via loadIdentity(). Return true if it has `ownId` field and no `npub` field.

       - Export `async function migrateOwnIdToNpub(): Promise<{npub?: string; migrated: boolean; queueReEncrypted?: number}>`:
         (a) Load old identity. If null or already has npub, return {migrated: false}.
         (b) Extract seed from old identity.
         (c) Call `importFromMnemonic(seed)` to derive NIP-06 keypair. IMPORTANT: This produces DIFFERENT keys than the old PBKDF2-derived keys (research pitfall 3). The old signing keys are discarded.
         (d) Generate browser-scoped CryptoKey via `generateBrowserScopedKey()`.
         (e) Encrypt {seed, nsec} via `encryptKeys()`.
         (f) Save CryptoKey via `saveBrowserKey()`.
         (g) Build EncryptedIdentityRecord with npub, protectionType 'none', iv, encryptedKeys, migratedFrom 'ownid'.
         (h) Save via `saveEncryptedIdentity()`.
         (i) Re-encrypt offline queue messages (per user decision: "re-encrypt offline queue messages"):
             - Call `loadAllQueuedMessages()` to get all pending QueuedMessages from IndexedDB
             - Filter to messages without `relayEventId` (not yet delivered to relay)
             - For each pending message:
               * Compute NIP-44 conversation key: `getConversationKey(newIdentity.privateKey, message.to)`
               * Re-encrypt the plaintext payload: `nip44Encrypt(message.payload, conversationKey)`
               * Update the message payload with the NIP-44 ciphertext
               * Save updated message back via `saveQueuedMessage(message)`
             - Track count of re-encrypted messages
             - Wrap in try/catch — queue re-encryption failure is non-fatal (log warning, continue migration)
         (j) Delete old identity via `clearIdentity()`.
         (k) Dispatch rootScope event 'nostra_identity_loaded' with {npub, displayName: null, nip05: null}.
         (l) Return {npub, migrated: true, queueReEncrypted: count}.

       - Wrap entire migration in try/catch. On error, do NOT delete old identity (safety). Log error and return {migrated: false}.

    3. Write migration tests with mocked IndexedDB. Test: successful migration flow, no-op when no old identity, no-op when already migrated, error recovery preserves old data, offline queue re-encryption (verify queued messages are NIP-44 encrypted and decodeable with new keypair via getConversationKey + nip44Decrypt).

    Note: Virtual peer re-mapping is NOT needed during migration because the virtual-peers-db maps OTHER users' pubkeys to peerIds. Only OUR identity changes. Confirm by reading virtual-peers-db.ts.
