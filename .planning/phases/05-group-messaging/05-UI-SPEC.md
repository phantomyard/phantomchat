# Phase 5 — UI Design Contract

> Visual and interaction contract for group messaging. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | tweb (Nostra.chat) built-in components |
| Icon library | tgico (project icon font, `src/scss/tgico.scss`) |
| Font | Roboto via `--font-regular`; monospace via `--font-monospace` |

**Note:** This project uses tweb's existing SCSS + CSS custom properties design system. No shadcn, Tailwind, or external component library. All new components follow tweb patterns: `.module.scss` scoped styles, `classNames()` helper, Solid.js `.tsx` components.

## Spacing Scale

Declared values (inherited from tweb CSS custom properties and SCSS variables):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element spacing |
| sm | 8px | Compact padding (`--chat-input-inner-padding: .5rem`) |
| md | 16px | Default padding, list item padding, bubble margin |
| lg | 24px | Section padding in sidebar, modal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Avatar size in group info header |
| 3xl | 64px | Page-level spacing (unused in this phase) |

Exceptions:
- Touch target minimum: 44px for member list action buttons (remove, admin toggle)
- Chat bubble padding: `13px` horizontal (`$chat-padding: .8125rem`), `8px` on handhelds — **tweb upstream constant, not modified by this phase**
- Avatar sizes: 48px in chat list rows, 120px in group info header (tweb standard)

## Color

All values inherit from tweb's theme system (`src/helpers/themeController.ts`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-color-true)` — light: `#f4f4f5`, dark: `#181818` | Chat background, app background |
| Secondary (30%) | `var(--surface-color)` — light: `#ffffff`, dark: `#212121` | Chat list rows, sidebar panels, group info card, message bubbles (incoming) |
| Accent (10%) | `var(--primary-color)` — light: `#3390ec`, dark: `#8774E1` | See reserved list below |
| Destructive | `var(--danger-color)` | Remove member button, Leave group button, destructive confirmations |

Accent reserved for:
- "Create Group" floating action button
- "Add Member" button in group info
- Unread badge count in chat list
- Selected/active states in member picker checkboxes
- Send button in chat input
- Online status dot on member avatars

**Sender name colors in group bubbles:** Use tweb's existing 7 avatar color palette (red `#FF845E`, orange `#FEBB5B`, violet `#B694F9`, green `#9AD164`, cyan `#53edd6`, blue `#5CAFFA`, pink `#FF8AAC`). Sender name color matches the sender's avatar color, assigned deterministically from their peer ID (existing tweb behavior via `--peer-color`).

## Interaction Contracts

### IC-01: Group Creation Flow

**Trigger:** User taps "New Group" button in chat list sidebar (or floating action button).

**Steps:**
1. **Contact Picker** — Full-screen slide-in from right. Checkboxes next to each contact. Selected contacts appear as chips at top. Counter shows "N/12 members". When N=12, remaining contacts become non-interactive (opacity 0.3). "Continue to Details" button in top-right (accent color, enabled when >= 1 contact selected).
2. **Group Details** — Slide-in from right. Large avatar placeholder (tap to pick photo from gallery). Text input for group name (required, max 128 chars). Optional description text area. "Create Group" button (accent color, enabled when name is non-empty).
3. **Created** — Navigates to the new group chat. Service message: "Group created".

**Back behavior:** Hardware/gesture back returns to previous step. Cancel at step 1 returns to chat list.

### IC-02: Group Chat List Entry

**Layout:** Identical to 1:1 chat list row (tweb standard).
- Left: Group avatar (48px circle). If no custom avatar, show `GroupAvatarInitials` with first letter of group name, background color derived from group ID hash using tweb avatar color palette.
- Center: Bold group name (top), preview text "SenderName: message text..." (bottom, secondary text color). If service message, show italic service text without sender prefix.
- Right: Timestamp (top), unread badge (bottom, accent background, white text).

### IC-03: Group Message Bubbles

**Focal point:** The primary chat view focal point is the most recent message cluster at the bottom of the viewport, anchored by the chat input bar. The user's eye naturally flows from the input bar upward through the latest messages.

**Incoming messages (from other members):**
- Sender name displayed above bubble text, 14px weight 500, colored with sender's peer avatar color.
- First message from a different sender in sequence shows the name; consecutive messages from same sender omit it (tweb standard grouping behavior).
- Avatar thumbnail (28px) shown to the left of the first bubble in a sender's sequence (tweb standard for group chats).

**Outgoing messages (own):**
- Same as 1:1 outgoing bubbles (green/purple tinted background depending on theme).
- No sender name shown (own messages are visually distinct by alignment + color).

**Delivery indicators:**
- Single check: sent to relay.
- Double check: delivered to ALL members.
- Blue double check: read by ALL members.
- Tap on delivery indicator: popup showing per-member delivery state (list of members with individual check/double-check/blue-check).

### IC-04: Service Messages

**Style:** Centered gray pill bubbles (existing `ServiceBubble` component). Text is 16px, weight 400, secondary text color on semi-transparent background.

**Copy for each event type:**

| Event | Service Message Text |
|-------|---------------------|
| Group created | "{CreatorName} created the group" |
| Member added | "{AdminName} added {MemberName}" |
| Member removed | "{AdminName} removed {MemberName}" |
| Member left | "{MemberName} left the group" |
| Name changed | "{AdminName} changed the group name to {NewName}" |
| Avatar changed | "{AdminName} changed the group photo" |
| Admin transferred | "{OldAdmin} made {NewAdmin} the group admin" |

### IC-05: Group Info Sidebar

**Trigger:** Tap group name/avatar in chat topbar.

**Layout (top to bottom):**
1. **Header:** Large avatar (120px), group name (18px bold), member count ("N members"), description if set (14px, secondary text color).
2. **Actions row:** Mute notifications toggle, Search messages button.
3. **Members section:** Header "Members" with count. Each row: avatar (40px) + name + role badge ("admin" pill, accent background, white text, 11px). Admin sees "Add Member" row at top (accent colored plus icon). Admin sees "Remove" context action on long-press/right-click of non-admin members.
4. **Shared media section:** Grid of shared photos/videos (reuse tweb shared media component).
5. **Danger zone:** "Leave Group" button (danger color text, no background). If admin: "Delete Group" button below.

### IC-06: Member Management

**Add member:**
- From GroupInfo sidebar, admin taps "Add Member" row.
- Contact picker opens (same as creation flow, minus existing members).
- Selected contacts receive `group_add_member` control message.
- Service message appears in chat.

**Remove member:**
- Admin taps member row in GroupInfo -> context menu with "Remove from Group" (danger color).
- Confirmation popup: "Remove {Name} from {GroupName}?"
- Buttons: "Keep Member" (secondary, default focus) | "Remove {Name}" (danger background, white text).
- On confirm: member removed, service message appears.

**Leave group:**
- From GroupInfo sidebar, user taps "Leave Group".
- Confirmation popup: "Leave {GroupName}? You will need a new invite to rejoin."
- Buttons: "Stay in Group" (secondary, default focus) | "Leave Group" (danger background, white text).
- On confirm: chat removed from list, `group_leave` control message sent.

### IC-07: Privacy Setting

**Location:** Settings > Privacy and Security > Groups.

**Layout:** Radio group with 3 options:
- "Everyone" (default) — Anyone can add you to groups.
- "My Contacts" — Only contacts can add you to groups.
- "Nobody" — Group invitations go to Message Requests.

**Style:** Standard tweb radio list (reuse existing privacy settings pattern from read receipt toggle).

## Responsive Behavior

Follows tweb's existing responsive breakpoints:

| Breakpoint | Behavior |
|------------|----------|
| Handhelds (`<= 600px`) | Full-screen panels. Group info replaces chat view. Member picker is full-screen. Chat padding reduces to 8px. Avatar in chat list: 48px. |
| Small screens (`<= 925px`) | Sidebar overlays chat. Group info slides over chat. |
| Desktop (`> 925px`) | Group info opens as right sidebar panel alongside chat (tweb standard `right-column`). |

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Focus management | Group creation steps trap focus within active step. Return focus to trigger on cancel. |
| Keyboard navigation | Member list navigable with arrow keys. Enter to select/deselect. Tab to action buttons. |
| Screen reader | Group avatar initials: `aria-label="{GroupName}"`. Member role announced: "{Name}, admin" vs "{Name}, member". Service messages announced via `aria-live="polite"`. |
| Touch targets | All interactive elements >= 44px touch target. Member list rows: 48px height. Action buttons: 44px minimum. |
| Color contrast | Sender name colors meet WCAG AA against white/dark bubble backgrounds (tweb avatar palette verified). Danger color (#DF3F40 range) meets AA on both themes. |
